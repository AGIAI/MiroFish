# MiroFish Quant Signal Implementation Plan

## Overview

Transform MiroFish from a swarm intelligence social simulation platform into a **world-class quant trading signal generator** for stocks, commodities, and crypto. The core insight: MiroFish already has the hardest part built — multi-agent simulation with LLM reasoning and knowledge graph memory. What it needs is domain-specific adaptation to financial markets and a quantitative output layer.

**Current Pipeline**: Documents → Knowledge Graph → Agent Personas → Social Simulation → Narrative Report

**Target Pipeline**: Market Data + News → Financial Knowledge Graph → Trader Archetypes → Market Microstructure Simulation → Calibrated Trading Signals → Bot API

```python
class OrderBookSimulator:

    def __init__(self, initial_price, initial_liquidity_depth):

        self.asks = SortedDict()    # price → quantity
        self.mid_price = initial_price

        """Walk up the ask side, consuming liquidity."""
        # Returns: fill_price, slippage, market_impact

    def market_sell(self, quantity, agent_id):
        """Walk down the bid side, consuming liquidity."""

    def limit_order(self, side, price, quantity, agent_id):
        """Add liquidity to the book."""

    def get_spread(self):
        """Current bid-ask spread."""

    def get_depth(self, levels=10):
```

Key dynamics to simulate:
- **Price impact**: Large orders move the price proportional to order size / liquidity
- **Slippage**: Market orders execute across multiple price levels
- **Spread dynamics**: Widens during volatility, narrows during calm
- **Liquidity replenishment**: Market makers re-enter after large moves
- **Cascade effects**: Stop-loss triggers causing waterfall liquidations

```python
            # Signal Components
            agent_consensus=0.68,               # % of agents bullish
            agent_conviction_strength=0.71,     # average conviction of majority
            narrative_momentum=0.75,            # narrative strength + direction
            flow_imbalance=0.62,                # net buy/sell pressure in sim
            regime="trending",                  # trending | ranging | volatile | crisis
            crowding_risk=0.30,                 # how one-sided the trade is
            information_edge_score=0.45,        # how much "new" info vs priced-in

            # Metadata
            simulation_rounds=72,
            agents_participated=150,
            dominant_narrative="ETF inflow acceleration",
            key_risk="FOMC rate decision in 48h",
            similar_historical_setups=[
                {"date": "2025-01-10", "outcome": "+3.2%", "similarity": 0.82},
                {"date": "2024-11-05", "outcome": "+5.1%", "similarity": 0.76},
            ],
        )
```

### Signal Component Extraction Logic

| Component | How It's Derived |
|-----------|-----------------|
| **Agent Consensus** | Count agents long vs short at simulation end |
| **Conviction Strength** | Average position size of majority-side agents |
| **Narrative Momentum** | Rate of change of SHARE_THESIS actions + sentiment direction |
| **Flow Imbalance** | (Total buy volume - total sell volume) / total volume |
| **Regime** | Volatility + autocorrelation of simulated price path |
| **Crowding Risk** | Herfindahl index of agent positions (concentration) |
| **Information Edge** | Delta between agent decisions before/after news injection |

## 6. Backtesting Framework

**Non-negotiable for any quant.** No backtest = no trust.

### New Service: `backend/app/services/backtester.py`

```python
class Backtester:
    """Walk-forward backtesting of simulation-generated signals."""

    def run_backtest(self, asset, start_date, end_date, config):
        """
        For each historical period:
        1. Collect market data + news up to time T
        2. Run simulation with that data
        3. Extract signal
        4. Record predicted vs actual outcome at T + horizon
        5. NEVER peek at future data
        """

    def calculate_metrics(self, results) -> BacktestMetrics:
        return BacktestMetrics(
            total_signals=len(results),
            win_rate=0.58,
            avg_win_pct=2.3,
            avg_loss_pct=-1.1,
            profit_factor=2.1,              # gross profit / gross loss
            sharpe_ratio=1.85,
            sortino_ratio=2.40,
            calmar_ratio=1.20,
            max_drawdown_pct=-12.3,
            max_drawdown_duration_days=18,
            avg_holding_period_hours=36,
            signal_decay_halflife_hours=8,  # how fast alpha degrades
            correlation_to_benchmark=0.15,  # low = good, means alpha is unique
            tail_ratio=1.3,                 # right tail / left tail
        )

    def monte_carlo(self, results, n_simulations=10000):
        """Shuffle trade outcomes to estimate probability of ruin."""

    def walk_forward_optimize(self, asset, params_grid):
        """Optimize on window N, test on window N+1, repeat."""
```

### Key Backtesting Principles

- **Walk-forward only**: Train on past, test on future, never the reverse
- **Transaction costs**: Include slippage, fees, funding rates
- **Survivorship bias**: Include delisted assets in historical data
- **Regime-aware**: Report metrics per regime (bull/bear/sideways)
- **Out-of-sample**: Always hold out 30% of data untouched until final validation
- **Multiple timeframes**: Test signals across 1h, 4h, 1d, 1w horizons

## 7. Regime Detection

Markets behave fundamentally differently in different regimes. The simulation must adapt.

### New Service: `backend/app/services/regime_detector.py`

| Regime | Detection Method | Agent Behavior Adjustment | Signal Adjustment |
|--------|-----------------|--------------------------|-------------------|
| **Low Vol Trending** | ADX > 25, ATR percentile < 30 | Momentum traders dominate | Increase conviction, wider stops |
| **High Vol Choppy** | ADX < 20, ATR percentile > 70 | All agents underperform | Reduce position size, tighten stops |
| **Narrative-Driven** | News volume spike + price divergence from indicators | Narrative agents dominate | Weight sentiment signals higher |
| **Liquidity Crisis** | Spread widening + volume collapse | Market makers withdraw | Widen spreads in sim, reduce size to minimum |
| **Mean-Reverting** | Hurst exponent < 0.5, high autocorrelation at lag-1 | Contrarians outperform | Fade extremes, tighter take-profits |
| **Breakout** | Bollinger squeeze + volume expansion | Momentum agents activate | Fast entry, trail stops |

### Implementation

```python
class RegimeDetector:
    def detect(self, price_data, volume_data, news_data) -> Regime:
        """
        Uses ensemble of:
        - Hidden Markov Model (volatility states)
        - Change-point detection (structural breaks)
        - Hurst exponent (trending vs mean-reverting)
        - ADX + ATR combination (trend strength + volatility)
        - News volume anomaly detection (narrative regime)
        """

    def adjust_simulation_params(self, regime, base_config) -> SimConfig:
        """Modify agent behaviors and simulation parameters for detected regime."""
```

## 9. Signal Calibration & Ensemble

### New Service: `backend/app/services/signal_calibrator.py`

```python
class SignalCalibrator:
    """Ensures signal conviction scores are well-calibrated."""

    def calibrate(self, raw_signal, historical_signals):
        """
        Bayesian calibration:
        - Prior: historical base rate for this setup type
        - Likelihood: simulation result strength
        - Posterior: calibrated conviction score

        When we say 70% conviction, it should win ~70% of the time.
        """

    def ensemble(self, signals_from_multiple_runs):
        """
        Run N simulations with different random seeds.
        Ensemble the results:
        - If 8/10 runs agree on direction → high conviction
        - If 5/10 agree → low conviction / flat
        - Variance across runs indicates uncertainty
        """

    def debias(self, signal, known_biases):
        """
        Correct for systematic biases:
        - Agents tend 60/40 bullish → subtract 10% from bullish consensus
        - Momentum agents overweight recent moves → decay their signal
        - Narrative agents overreact to news → dampen their contribution
        """

    def brier_score(self, predicted_probabilities, actual_outcomes):
        """Track calibration quality over time."""
```

### Ensemble Strategy

For each signal generation:
1. Run 5-10 simulations with different random seeds
2. Vary agent compositions slightly (±10% of each archetype)
3. Vary information lag (some agents get news 1min early, some 5min late)
4. Take the median direction and mean conviction
5. Use inter-run variance as an uncertainty estimate

## 10. Risk Management Layer

### New Service: `backend/app/services/risk_manager.py`

```python
class RiskManager:
    """Portfolio-level risk management for signal output."""

    def size_position(self, signal, portfolio_state):
        """
        Kelly Criterion (half-Kelly for safety):
        f* = (p * b - q) / b
        where p = win_prob, q = 1-p, b = avg_win/avg_loss

        Then adjust for:
        - Current drawdown (reduce size when in drawdown)
        - Correlation to existing positions
        - Volatility regime (reduce in high vol)
        - Concentration limits
        """

    def check_portfolio_constraints(self, new_signal, existing_positions):
        """
        Enforce:
        - Max 5% portfolio risk per signal
        - Max 20% total portfolio risk
        - Max 3 correlated positions (correlation > 0.7)
        - No new signals during circuit breaker
        """

    def circuit_breaker(self, portfolio_state):
        """
        Stop generating signals when:
        - Daily drawdown > 3%
        - Weekly drawdown > 7%
        - 5 consecutive losses
        - Regime detection shows crisis mode
        """

    def calculate_stop_loss(self, signal, price_data):
        """
        ATR-based stops:
        - Trending regime: 2.5x ATR
        - Ranging regime: 1.5x ATR
        - High vol: 3.0x ATR
        Never risk more than 2% of portfolio per trade.
        """
```

## Key Technical Decisions

### LLM Usage in the Quant Pipeline

The LLM remains central but its role shifts:

1. **Ontology Generation** → Financial entity extraction from news/filings
2. **Agent Decision Making** → Each trader agent uses LLM to reason about market conditions given their archetype constraints
3. **Narrative Analysis** → LLM synthesizes news into narrative scores
4. **Report Generation** → Explains signal rationale in human-readable format
5. **Interview Mode** → "Ask the market" — query why agents are positioned the way they are

### What NOT to Use LLM For

- Price prediction directly (LLMs are not good at this)
- Technical indicator calculation (use numpy/pandas)
- Order matching (deterministic algorithm)
- Risk calculations (pure math)
- Backtest execution (pure computation)

The LLM's role is **qualitative reasoning within a quantitative framework** — exactly where it excels.

### Edge Over Traditional Quant Signals

| Traditional Quant | MiroFish Quant |
|-------------------|----------------|
| Single model, single perspective | 150+ agents with diverse strategies |
| Static rules | Adaptive reasoning per market condition |
| No narrative understanding | LLM-powered narrative analysis |
| Backward-looking indicators | Forward-looking agent simulations |
| No institutional memory | Knowledge graph retains all market history |
| Black box | Explainable (interview any agent for rationale) |
